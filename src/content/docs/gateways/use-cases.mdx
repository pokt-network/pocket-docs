Teams building on Pocket Network's gateway infrastructure fall into four patterns:

## 1. Commercial RPC Provider

Build a managed RPC service (like Infura or Alchemy) powered by Pocket's decentralized supplier pool. Set your own pricing, issue API keys, and keep the margin between customer revenue and protocol cost.

**Why Pocket:** Lower infrastructure cost than running your own nodes, built-in multi-chain support, decentralized redundancy without managing it yourself.

## 2. dApp Infrastructure

Run a gateway dedicated to your own application's traffic. Instead of paying a centralized provider per request, stake POKT and route your traffic through the network directly.

**Why Pocket:** Predictable costs (proportional to usage), no vendor lock-in, censorship resistance for critical infrastructure.

## 3. AI Agent Data Layer

Route AI agent traffic through Pocket — blockchain queries, DeFi data, multi-chain state. [MCP integration](/developers/ai-agents) makes Pocket a natural fit for agent frameworks.

**Why Pocket:** 60+ chains through one interface, no API key management for agent fleets, [x402](/developers/ai-agents) autonomous payment support (roadmap).

## 4. Chain Foundation Partnership

Chain foundations sponsor Pocket endpoints for their developer communities. PNF provides dedicated, unlimited access at wholesale (protocol burn cost only).

**Why Pocket:** Non-profit pricing, decentralized infrastructure, ready-made 60+ chain ecosystem. Contact [directors@pokt.foundation](mailto:directors@pokt.foundation).
